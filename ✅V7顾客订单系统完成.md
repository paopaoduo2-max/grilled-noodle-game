# ✅ V7 顾客订单系统完成

## 🎯 实现每个顾客对应一个订单系统

本次更新实现了完整的顾客订单系统，每个顾客都有独立的订单，订单需求显示在顾客头顶。

---

## 📋 核心功能

### ✅ 1. 多顾客订单系统

**数据结构：**
```typescript
// 顾客列表，每个顾客包含：
// - node: 顾客节点
// - order: 顾客的订单（CustomerOrder）
// - orderLabel: 订单UI标签（显示在头顶）
private customers: Array<{
    node: Node, 
    order: CustomerOrder | null, 
    orderLabel: Node | null
}> = [];
```

**初始化流程：**
```
1. setupCustomers() - 查找所有Customer节点
2. generateOrdersForAllCustomers() - 为每个顾客生成订单
3. createOrderUI() - 在顾客头顶创建订单UI
```

---

### ✅ 2. 订单UI显示在顾客头顶

**创建订单UI：**
```typescript
// 在顾客节点上方创建订单标签
const orderLabelNode = new Node(`OrderLabel_${index}`);
const label = orderLabelNode.addComponent(Label);

// 设置位置（顾客头顶）
orderLabelNode.setPosition(0, 80, 0);  // 在顾客节点上方80px

// 添加到顾客节点
customer.node.addChild(orderLabelNode);
```

**订单显示格式：**
```
📋 订单：
✅ 🥚 鸡蛋
✅ 🌿 香菜
✅ 🧅 洋葱

不要：
❌ 🌭 香肠
```

---

### ✅ 3. 出餐到指定顾客

**出餐流程：**
```
1. 点击出餐按钮
2. 找到第一个有订单的顾客
3. 将食物送到该顾客
4. 评估订单
5. 清除订单并生成新订单
```

**坐标转换：**
```typescript
// 获取顾客节点的世界坐标
const customerWorldPos = new Vec3();
customer.node.getWorldPosition(customerWorldPos);

// 转换为Canvas本地坐标
const servePosition = new Vec3(
    customerWorldPos.x - canvasWorldPos.x,
    customerWorldPos.y - canvasWorldPos.y + 30,
    0
);
```

---

### ✅ 4. 订单评估和完成

**评估流程：**
```typescript
evaluateOrderForCustomer(food, customer) {
    1. checkOrder(food, customer.order) - 检查订单匹配度
    2. completeOrder(score) - 完成订单并结算
    3. 清除顾客订单
    4. 生成新订单
    5. 更新订单UI
}
```

**评分规则：**
```
✅ 完美匹配：100%分数
⚠️ 缺少食材：50%分数
❌ 添加了不要的食材：30%分数
🔥 烧焦：基础分数降低
```

### 完整流程
```

   → setupCustomers() 找到所有顾客节点
   → generateOrdersForAllCustomers() 为每个顾客生成订单

   → 点击出餐按钮

   → 食物动画移动到顾客位置
   → evaluateOrderForCustomer() 评估订单
   → 清除订单，生成新订单，更新UI

4. 循环
   → 每个顾客都有独立订单
   → 订单完成后自动生成新订单
```
