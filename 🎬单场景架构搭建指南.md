## 🔧 需要手动完成的步骤

### 步骤1：在Cocos Creator中打开GameScene

1. 在 **资源管理器** 中找到 `assets/Scenes/GameScene.scene`
2. 双击打开场景

### 步骤2：绑定CameraManager组件

1. 在 **层级管理器** 中选择 `CameraManager` 节点
2. 在 **属性检查器** 中点击 **添加组件** → **自定义脚本** → **CameraManager**
3. 将以下节点拖拽到对应属性：
   - `MenuCamera` → menuCamera
   - `PrepareCamera` → prepareCamera
   - `CookingCamera` → cookingCamera
   - `ResultCamera` → resultCamera
   - `MenuCanvas` → menuCanvas
   - `PrepareCanvas` → prepareCanvas
   - `CookingCanvas` → cookingCanvas
   - `ResultCanvas` → resultCanvas

### 步骤3：复制MainMenu UI内容

#### 方法A：从旧场景复制（推荐）
1. 打开 `assets/Scenes/MainMenu.scene`
2. 在 **层级管理器** 中选中 `Canvas` 下的所有UI节点（除了Canvas本身）
3. 按 `Ctrl+C` 复制
4. 切换到 `GameScene.scene`
5. 选中 `MenuCanvas` 节点
6. 按 `Ctrl+V` 粘贴
7. 调整位置和UI脚本组件绑定

#### 方法B：手动创建（如果复制有问题）
在 `MenuCanvas` 下手动创建以下节点结构：
```
MenuCanvas
├── Background (Sprite)
├── TitleLabel (Label) - 已创建
├── LevelButtonsPanel (Layout) - 已创建
│   └── （这里会动态创建关卡按钮）
```

**重要：** 为 `MenuCanvas` 添加 `MainMenuUI` 组件，并设置：
- `levelPanel`: 绑定到 `LevelButtonsPanel`

1. 打开 `assets/Scenes/ResultScene.scene`
2. 复制 `Canvas` 下的所有UI节点
3. 粘贴到 `GameScene` 的 `ResultCanvas` 下
4. 为 `ResultCanvas` 添加 `ResultUI` 组件
5. 绑定相关属性

### 步骤7：配置Camera可见层（可选，用于优化性能）

如果想通过Layer来完全隔离不同UI：

1. 在 **项目设置** → **Layers** 中创建自定义层：
   - Layer 20: UI_Menu
   - Layer 21: UI_Prepare
   - Layer 22: UI_Cooking
   - Layer 23: UI_Result

2. 为每个Canvas及其子节点设置对应Layer：
   - MenuCanvas 及子节点 → UI_Menu (Layer 20)
   - PrepareCanvas 及子节点 → UI_Prepare (Layer 21)
   - CookingCanvas 及子节点 → UI_Cooking (Layer 22)
   - ResultCanvas 及子节点 → UI_Result (Layer 23)

3. 为每个Camera设置对应的Visibility：
   - MenuCamera → 只勾选 UI_Menu
   - PrepareCamera → 只勾选 UI_Prepare
   - CookingCamera → 只勾选 UI_Cooking
   - ResultCamera → 只勾选 UI_Result

### 步骤8：更新项目设置

1. 打开 **项目** → **项目设置** → **引擎模块** → **场景资源列表**
2. 将 `GameScene` 设置为 **启动场景**
3. 保存设置

### 步骤9：测试

1. 点击 **预览** 按钮
2. 应该能看到主菜单界面
3. 选择关卡并点击"开始游戏"
4. 应该切换到准备阶段（不加载新场景，只切换Camera）

## 🎯 单场景架构的优势

### 1. 性能优化
- ✅ **无场景切换开销**：不需要卸载/加载场景
- ✅ **即时切换**：Camera切换是瞬间的，无延迟
- ✅ **内存稳定**：所有UI一次性加载，避免反复加载

### 2. 数据共享
- ✅ **跨阶段数据**：各个阶段共享同一个场景，数据传递更简单
- ✅ **全局管理器**：GameManager、CameraManager等无需处理场景切换

### 3. 开发便利
- ✅ **统一管理**：所有UI在一个场景中，方便查看和调试
- ✅ **快速测试**：可以在编辑器中直接切换Camera预览不同界面

### 4. UI交互
- ✅ **跨界面动画**：可以实现界面间的过渡动画
- ✅ **共享元素**：可以有跨界面的UI元素（如通知系统）

## 🔍 如何工作

### CameraManager的切换逻辑

```typescript
// 切换到某个界面时：
1. 禁用所有Camera
2. 隐藏所有Canvas
3. 启用目标Camera
4. 显示目标Canvas
```

### GameManager的集成

```typescript
// 例如：进入准备阶段
enterPreparePhase() {
    // 1. 更新游戏状态
    this._currentPhase = GamePhase.PREPARE;
    
    // 2. 通过CameraManager切换视图（不加载场景）
    CameraManager.getInstance().switchToPrepare();
    
    // 3. 如果CameraManager不存在，回退到场景切换
    // （向后兼容）
}
```

## ⚠️ 注意事项

### 1. UI脚本适配
某些UI脚本可能需要调整节点查找路径，因为：
- 旧架构：UI脚本在 `Canvas` 上
- 新架构：UI脚本在 `MenuCanvas/PrepareCanvas` 等上

### 2. 事件监听
确保各个UI脚本的 `onEnable()` 和 `onDisable()` 正确处理事件监听：
```typescript
onEnable() {
    // 重新注册事件
}

onDisable() {
    // 取消注册事件
}
```

### 3. 性能监控
虽然单场景架构避免了场景切换，但会占用更多初始内存。如果内存吃紧，可以考虑：
- 懒加载某些Canvas的内容
- 使用对象池管理UI元素

## 📝 问题排查

### 问题1：Camera切换后看不到UI
**检查：**
- Canvas的 `active` 状态是否正确
- Camera的 `enabled` 状态是否正确
- Camera的 `orthoHeight` 是否为360
- Canvas的 `position` 是否为(0,0,0)

### 问题2：UI按钮不响应
**检查：**
- Canvas上的 `Canvas` 组件是否存在
- UI脚本组件是否正确绑定
- 节点引用是否都已绑定

### 问题3：界面布局错乱
**检查：**
- 复制UI时可能丢失某些组件属性
- 手动检查 `UITransform`、`Widget`、`Layout` 等组件配置
