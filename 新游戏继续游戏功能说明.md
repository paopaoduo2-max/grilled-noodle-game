# 新游戏 / 继续游戏功能说明

## 功能概述
在主菜单添加了"新游戏"和"继续游戏"的区分功能：
- **新游戏**：显示教程，老王会引导玩家学习
- **继续游戏**：跳过教程，直接进入游戏（测试阶段）

## 修改内容

### 1. GameManager.ts
- 添加 `_skipTutorial` 标志
- 修改 `startLevel(levelId, skipTutorial)` 方法，接收跳过教程参数
- 添加 `skipTutorial` getter 供其他脚本访问

```typescript
// 开始关卡
public startLevel(levelId: number, skipTutorial: boolean = false) {
    this._skipTutorial = skipTutorial;
    console.log('跳过教程:', skipTutorial);
    // ...
}

public get skipTutorial(): boolean { 
    return this._skipTutorial; 
}
```

### 2. MainMenuUI.ts
- **新游戏按钮**：调用 `gameManager.startLevel(1, false)` - 显示教程
- **继续游戏按钮**：调用 `gameManager.startLevel(1, true)` - 跳过教程
- 测试阶段：继续游戏按钮始终可用

```typescript
// 新游戏
public onStartGame() {
    gameManager.startLevel(1, false);  // 显示教程
}

// 继续游戏（测试阶段）
public onContinueGame() {
    gameManager.startLevel(1, true);  // 跳过教程
}
```

### 3. TutorialManager.ts
- 导入 `GameManager`
- 在 `startTutorial()` 开始时检查 `gameManager.skipTutorial`
- 如果为 true，直接返回，不启动教程

```typescript
public startTutorial() {
    const gameManager = GameManager.Instance;
    if (gameManager && gameManager.skipTutorial) {
        console.log('⏭️ GameManager 设置跳过教程');
        this.isActive = false;
        return;
    }
    // ... 启动教程
}
```

## 使用说明

### 开发测试
1. 点击"新游戏" → 老王出现，显示教程
2. 点击"继续游戏" → 直接进入游戏，无教程

### 未来正式版本
需要修改 `MainMenuUI.ts` 中的 `updateContinueButton()` 和 `onContinueGame()`：

```typescript
// 恢复存档检查逻辑
private updateContinueButton() {
    if (!this.continueBtn) return;
    const latestSave = SaveManager.getLatestSave();
    this.continueBtn.interactable = latestSave !== null;
}

// 恢复从存档加载
public onContinueGame() {
    const latestSave = SaveManager.getLatestSave();
    if (!latestSave) return;
    this.loadGameFromSave(latestSave);
}
```

## 注意事项
1. 测试阶段"继续游戏"直接跳过教程进入关卡
2. 正式版需要恢复存档加载逻辑
3. 教程完成后会自动保存到 localStorage
4. 可以通过 `TutorialManager.resetTutorial()` 重置教程状态
