# 🚀 快速开始指南

**欢迎使用《东北料理王》游戏框架！**

这份指南将帮助你在**10分钟内**完成第一次测试运行。

---

## 📋 前置要求

- ✅ Cocos Creator 3.8.7 已安装
- ✅ 项目已在Cocos Creator中打开
- ✅ 所有脚本已编译（无错误）

## ⚡ 第一步：验证框架完整性（2分钟）

### 1. 检查文件是否存在

打开文件管理器，确认以下文件存在：

```
✅ assets/Scripts/Data/GameConfig.ts
✅ assets/Scripts/Manager/ReviewManager.ts
✅ assets/Scripts/Manager/AchievementManager.ts
✅ assets/Scripts/Game/StepOperationSystem.ts
✅ assets/Scripts/UI/TutorialUI.ts
✅ assets/Scenes/TutorialScene.scene
```

### 2. 在Cocos Creator中检查场景

- 打开 **PrepareScene.scene**
- 在层级管理器中，找到Canvas节点
- 检查是否有 **ItemShopPanel** 节点 ✅

- 打开 **CookingScene.scene**
- 检查是否有以下节点：
  - ✅ ReviewManager
  - ✅ ReviewPanel
  - ✅ StepOperationPanel

### 3. 检查编译状态

- 查看Cocos Creator控制台
- 确保没有红色错误信息
- 如果有警告（黄色），可以暂时忽略

1. 在Cocos Creator中
2. 双击 `assets/Scenes/MainMenu.scene`
3. 场景会在编辑器中打开

### 2. 运行游戏

1. 点击顶部工具栏的 **运行** 按钮（▶️）

3. 浏览器会自动打开

### 3. 检查主菜单

在浏览器中，你应该看到：


## 🔧 常见问题排查

### 问题1：ItemShopPanel不显示

**症状**：准备场景中没有看到道具商店

**解决方案**：
1. 在Cocos Creator中打开PrepareScene
2. 确认ItemShopPanel节点存在
3. 选中Canvas节点
4. 在Inspector中找到PreparePhaseUI组件
5. 检查`itemShopPanel`属性是否绑定
6. 如果没绑定，拖拽ItemShopPanel到该属性
7. 保存场景并重新运行

### 问题2：控制台显示找不到GameManager

**症状**：控制台错误 `GameManager.Instance is null`

**解决方案**：
1. 确保从MainMenu.scene开始运行游戏
2. 不要直接运行PrepareScene或CookingScene
3. MainMenu中的GameManager节点会被设置为持久节点

### 问题3：脚本编译错误

**症状**：Cocos Creator控制台显示红色错误

**解决方案**：
1. 点击控制台的错误信息查看详情
2. 双击错误可以跳转到代码位置
3. 常见问题：
   - 缺少导入语句 → 添加 `import`
   - 类型不匹配 → 检查变量类型
4. 修复后保存文件，Cocos会自动重新编译

### 问题4：购买道具后没反应

**症状**：点击道具购买按钮，金币扣了但效果不明显

**解决方案**：
- 这是**正常的**！道具效果在CookingScene中生效
- 道具数据已保存到GameManager
- 在CookingScene中，道具效果会自动应用
- 目前部分道具效果需要在后续开发中实现
