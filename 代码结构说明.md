# 代码结构说明

## 架构概览

本项目采用**分层架构**设计，清晰分离数据、逻辑和UI层。

```
┌─────────────────────────────────────┐
│          UI Layer (UI层)            │
│   MainMenuUI, PrepareUI, etc.       │
├─────────────────────────────────────┤
│       Game Logic Layer (逻辑层)      │
│  CustomerManager, CookingSystem     │
├─────────────────────────────────────┤
│      Manager Layer (管理层)         │
│         GameManager (单例)           │
├─────────────────────────────────────┤
│       Data Layer (数据层)            │
│   GameConfig, Enums, Interfaces     │
└─────────────────────────────────────┘
```

## 文件详解

### 📁 Data/GameConfig.ts

**职责：** 游戏静态配置数据中心

**包含内容：**

1. **枚举定义**
   ```typescript
   GamePhase        // 游戏阶段
   IngredientType   // 食材类型
   CustomerType     // 客户类型
   OrderStatus      // 订单状态
   ```

2. **数据接口**
   ```typescript
   IngredientData   // 食材数据
   RecipeData       // 菜谱数据
   CustomerData     // 客户数据
   OrderData        // 订单数据
   LevelData        // 关卡数据
   ```

3. **静态配置**
   ```typescript
   GameConfig.INGREDIENTS_CONFIG  // 所有食材配置
   GameConfig.LEVELS              // 所有关卡配置
   GameConfig.CUSTOMER_TYPES      // 客户类型配置
   GameConfig.CUSTOMER_NAMES      // 客户名字库
   ```

**使用示例：**
```typescript
import { GameConfig, IngredientType } from '../Data/GameConfig';

// 获取食材配置
const noodleConfig = GameConfig.INGREDIENTS_CONFIG[IngredientType.NOODLE];
console.log(noodleConfig.name);  // "面饼"
console.log(noodleConfig.price); // 2

// 获取关卡数据
const level1 = GameConfig.LEVELS[0];
console.log(level1.levelName);   // "第一关：烤冷面摊位"
```

### 📁 Manager/GameManager.ts

**职责：** 全局游戏状态管理（单例模式）

**核心功能：**

1. **游戏状态管理**
   ```typescript
   private _currentLevel: LevelData
   private _currentPhase: GamePhase
   private _playerMoney: number
   private _ingredients: Map<IngredientType, number>
   ```

2. **场景流程控制**
   ```typescript
   startLevel(levelId: number)        // 开始关卡
   enterPreparePhase()                // 进入准备阶段
   enterCookingPhase()                // 进入制作阶段
   enterResultPhase()                 // 进入结算阶段
   returnToMenu()                     // 返回主菜单
   ```

3. **资源管理**
   ```typescript
   buyIngredient(type, count): boolean
   useIngredient(type, count): boolean
   getIngredientCount(type): number
   ```

4. **数据统计**
   ```typescript
   completeOrder(money, score)        // 订单完成
   orderTimeout()                     // 订单超时
   sessionStats                       // 会话统计
   ```

5. **数据持久化**
   ```typescript
   saveGameData()                     // 保存游戏
   loadGameData()                     // 加载游戏
   ```

**使用示例：**
```typescript
const gameManager = GameManager.Instance;

// 检查是否有足够食材
if (gameManager.hasEnoughIngredients('grilled_cold_noodle')) {
    // 开始制作
}

// 获取当前金币
console.log('金币:', gameManager.playerMoney);
```

**重要提示：**
- GameManager在MainMenu场景中创建，并设置为持久节点
- 在其他场景中通过`GameManager.Instance`访问
- 不要在其他场景重复创建GameManager节点

### 📁 Game/CookingSystem.ts

**职责：** 烹饪任务管理

**核心功能：**

1. **烹饪任务**
   ```typescript
   startCooking(orderId, recipe)      // 开始烹饪
   finishTask(taskId): boolean        // 完成任务
   getActiveTasks(): CookingTask[]    // 获取活跃任务
   ```

2. **资源检查**
   ```typescript
   // startCooking内部会：
   // 1. 检查是否有空位
   // 2. 检查食材是否足够
   // 3. 消耗食材
   // 4. 创建烹饪任务
   ```

3. **进度管理**
   ```typescript
   update(deltaTime)                  // 更新烹饪进度
   // task.progress: 0-1
   // task.cookTime: 剩余时间
   ```

**数据结构：**
```typescript
interface CookingTask {
    taskId: string
    orderId: string
    recipe: RecipeData
    state: CookingState       // idle, cooking, done
    progress: number          // 0-1
    cookTime: number          // 剩余时间
}
```

**使用流程：**
```typescript
// 1. 初始化
cookingSystem.init(3);  // 最多3个烹饪位

// 2. 开始烹饪
const result = cookingSystem.startCooking(orderId, recipe);
if (result.success) {
    console.log('开始烹饪', result.taskId);
}

// 3. 等待完成（监听事件）
cookingSystem.node.on('cooking-done', (task) => {
    // 烹饪完成，交付订单
    cookingSystem.finishTask(task.taskId);
});
```

## 数据流图

### 准备阶段数据流
```
用户点击购买
  ↓
PreparePhaseUI.onBuyIngredient()
  ↓
GameManager.buyIngredient()
  ↓
检查金币 → 扣除金币 → 增加食材
  ↓

  ↓

```

```
CustomerManager每8秒
  ↓
生成客户 → 创建订单
  ↓

  ↓
CookingPhaseUI.onCustomerArrived()
  ↓
创建订单卡片UI
  ↓
用户点击"开始制作"
  ↓
CookingPhaseUI.startCooking()
  ↓
CookingSystem.startCooking()
  ↓
检查资源 → 消耗食材 → 创建任务
  ↓

  ↓
触发 'cooking-done' 事件
  ↓
CookingPhaseUI.completeOrder()
  ↓
CustomerManager.completeOrder()
  ↓
计算收益 → GameManager.completeOrder()
  ↓
更新金币和评分
  ↓
移除订单UI
```

## 设计模式

### 1. 单例模式（Singleton）
**GameManager** 使用单例模式，确保全局只有一个实例。

```typescript
private static _instance: GameManager = null;

public static get Instance(): GameManager {
    return this._instance;
}

onLoad() {
    if (GameManager._instance === null) {
        GameManager._instance = this;
        director.addPersistRootNode(this.node);
    } else {
        this.node.destroy();
    }
}
```

### 2. 观察者模式（Observer）
使用Cocos Creator的事件系统实现观察者模式。

```typescript
// 发布者
this.node.emit('customer-arrived', order);

// 订阅者
node.on('customer-arrived', (order) => {
    // 处理事件
});
```

### 3. 组件模式（Component）
每个UI控制器都是独立组件，可复用和组合。

```typescript
@ccclass('PreparePhaseUI')
export class PreparePhaseUI extends Component {
    // 组件逻辑
}
```

### 4. 工厂模式（Factory）
动态创建UI元素和游戏对象。

```typescript
private createIngredientItem(type, name, price, color): Node {
    const itemNode = new Node('Ingredient');
    // 创建并配置节点
    return itemNode;
}
```

## 扩展建议

### 添加新功能的步骤

1. **数据层**
   - 在`GameConfig.ts`中添加配置数据
   - 定义新的接口和枚举

2. **逻辑层**
   - 在`Game/`文件夹创建新系统
   - 实现核心逻辑
   - 使用事件与其他系统通信

3. **管理层**
   - 在`GameManager`中添加状态管理
   - 提供统一的访问接口

4. **UI层**
   - 创建新的UI控制器
   - 监听逻辑层事件
   - 更新显示

### 代码规范建议

1. **命名规范**
   - 类名：PascalCase（如：`GameManager`）
   - 方法名：camelCase（如：`startLevel`）
   - 私有成员：下划线前缀（如：`_playerMoney`）
   - 常量：UPPER_CASE（如：`MAX_CUSTOMERS`）

2. **注释规范**
   ```typescript
   /**
    * 方法说明
    * @param参数说明
    * @returns 返回值说明
    */
   ```

3. **日志规范**
   ```typescript
   console.log('[模块名] 信息');
   console.warn('[模块名] 警告');
   console.error('[模块名] 错误');
   ```

## 性能优化建议

1. **对象池**
   - 订单卡片UI可使用对象池复用
   - 减少频繁创建销毁

2. **事件清理**
   - 在`onDestroy()`中移除事件监听
   - 防止内存泄漏

3. **UI更新优化**
   - 使用脏标记，只在数据改变时更新
   - 避免每帧更新所有UI

4. **资源管理**
   - 使用Resources动态加载
   - 及时释放不用的资源
