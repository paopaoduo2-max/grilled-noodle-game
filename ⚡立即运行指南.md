## 🎮 游戏测试流程

### 预期游戏流程

**1. 主菜单界面**
- 看到"东北料理王"标题
- 看到"第一关：烤冷面摊位"按钮（蓝色）
- 看到"第二关：待定"按钮（灰色，未解锁）
- 点击第一关，然后点击"开始游戏"按钮

**2. 准备阶段（60秒）**
- 显示金币：50
- 显示倒计时
- 看到5种食材（面饼、鸡蛋、香肠、香菜、酱料）
- 每种食材显示价格和数量
- 点击"购买 x1"按钮购买食材
  - 金币减少
  - 数量增加
- 购买足够食材后，点击"开始营业"

**3. 制作阶段（180秒）**
- 顶部显示：金币、时间、评分、客户数
- 约8秒后，左侧出现第一个客户订单
  - 显示客户名字
  - 显示"烤冷面"
  - 显示耐心进度条（绿色→黄色→红色）
- 点击订单的"开始制作"按钮
  - 食材自动消耗
  - 按钮变为"制作中..."
  - 5秒后自动完成
  - 获得金币和评分
  - 客户数+1
  - 订单消失
- 继续服务更多客户
- 时间结束或点击"结束营业"

**4. 结算界面**
- 显示"关卡通过"或"关卡失败"
- 显示赚取金币 vs 目标金币（200）
- 显示完成客户 vs 目标客户（15）
- 显示总评分和统计信息
- 点击按钮：
  - "重试" - 重新开始当前关卡
  - "下一关" - 进入下一关（通过后显示）
  - "返回菜单" - 回到主菜单

## ⚠️ 可能的问题和解决

### 问题1: 点击预览没有反应
**解决：**
```
1. 检查控制台是否有红色错误
2. 菜单 -> 开发者 -> 编译脚本
3. 重新点击预览
```

### 问题2: 场景切换失败
**解决：**
```
1. 菜单 -> 项目 -> 项目设置 -> 预览运行设置
2. 确认"启动场景"设置为 MainMenu
3. 在"构建发布"标签中，确认所有4个场景都在"参与构建场景"列表中
```

### 问题3: UI显示不正确
**这是正常的！** 原因：
- 脚本的@property引用需要手动绑定
- 不影响基本游戏逻辑
- 可以在编辑器中手动绑定引用来完善

**如何手动绑定（可选）：**
```
例如在PrepareScene中：
1. 打开 PrepareScene.scene
2. 选中 Canvas 节点
3. 在属性检查器中找到 PreparePhaseUI 组件
4. 将对应的节点拖入对应的属性框：
   - Ingredient Panel → 拖入 IngredientPanel 节点
   - Money Label → 拖入 MoneyLabel 节点
   - Timer Label → 拖入 TimerLabel 节点
   - Level Name Label → 拖入 LevelNameLabel 节点
   - Start Cooking Btn → 拖入 StartCookingBtn 节点
5. 保存场景（Ctrl+S）
```

### 问题4: 游戏逻辑不工作
**解决：**
```
1. 打开浏览器控制台（F12）
2. 查看 Console 标签
3. 搜索 [GameManager]、[CustomerManager] 等日志
4. 根据错误信息定位问题
```
