## 🎯 项目概述

这是一个**烤冷面制作模拟游戏**，玩家需要：
1. 点击食材按钮获取食材
2. 在面饼上按正确顺序添加食材（鸡蛋→翻面→其他食材）
3. 根据顾客订单制作正确的烤冷面
4. 将制作好的食物拖到打包盒中
5. 完成订单获得金币和评价

## 🎮 核心游戏系统

### 1. 食材系统

#### 食材类型 (`IngredientType`)
```typescript
DOUGH              // 面饼（点击烤架生成）
EGG                // 鸡蛋（最多3个）
CILANTRO           // 香菜（最多5个）
ONION              // 洋葱（最多5个）
SAUSAGE            // 香肠（最多5个，必放）
CHILI              // 辣椒（调料，最多3次）
SUGAR              // 白糖（调料，最多3次）
VINEGAR            // 醋（调料，最多3次）
GRILLED_NOODLE_SAUCE  // 烤冷面酱（必须，使用sauceBtn）
```

#### 食材交互逻辑
- **实体食材**（鸡蛋、洋葱、香菜、香肠）：
  - 连续点击可拿多个（鸡蛋max 3，其他max 5）
  - 点击面饼时，一次只添加一个
  - 鼠标右键或拿超过3个时自动放下
  
- **调料**（辣椒、白糖、醋、烤冷面酱）：
  - 点击拿起，可多次添加到面饼（max 3次/调料/面饼）
  - 再次点击按钮可放回

#### 食材跟随鼠标
- 点击食材按钮后，食材图片立即跟随鼠标
- 使用 `mouseFollower` 节点显示当前手持的食材
- 显示数量：实体食材显示 "emoji x数量"，调料只显示 "emoji"

### 2. 面饼制作流程

#### 面饼状态 (`DoughState`)
```typescript
RAW          // 生面饼（未加鸡蛋）
EGG_ADDED    // 已加鸡蛋（未翻面）
FLIPPED      // 已翻面（可以加其他食材）
BURNT        // 烤焦了（仍可操作：丢弃、移动、添加食材）
DONE         // 完成
```

#### 制作流程（严格顺序）
1. **点击烤架** → 生成生面饼（RAW状态）
2. **必须先加鸡蛋** → 面饼变为 EGG_ADDED 状态
3. **点击面饼翻面** → 面饼变为 FLIPPED 状态
4. **添加其他食材** → 洋葱、香菜、香肠、调料等
5. **注意**：如果未加鸡蛋就尝试加其他食材，会提示错误

#### 面饼位置系统
- 烤架有3个位置（索引0, 1, 2）
- 新面饼会循环放置在第一个可用位置
- 面饼位置：`xOffset - 250 + posIndex * 150`（向左偏移100px）
- 面饼移除后，其他面饼**不移动**（保持原位）

### 3. 订单系统

#### 订单结构 (`CustomerOrder`)
```typescript
interface CustomerOrder {
    id: number;
    eggCount: number | null;           // 鸡蛋数量（1/2/3），null=默认1个
    sausageCount: number | null;       // 香肠数量（1/2），null=默认1根
    extraIngredients: IngredientType[]; // 多放的食材（洋葱、香菜）
    extraCondiments: IngredientType[];  // 多加的调料（辣椒、白糖、醋）
    flavor: FlavorType;                 // 口味（甜辣/酸甜辣/酸甜）
    excludes: IngredientType[];         // 不要的食材（洋葱、香菜）
    reward: number;
    timeLimit: number;
    patience: number;
}
```

#### 订单显示格式
```
📋 订单：
加2个🥚              ← 只有≥2个蛋才显示
2根🌭                ← 只有≥2根香肠才显示
多放🧅洋葱            ← 额外食材
多加🌶️辣椒           ← 额外调料
不要🌿香菜            ← 排除的食材
甜辣                 ← 口味（最后显示）
```

#### 默认规则
- **默认食材**：1个鸡蛋 + 1根香肠 + 洋葱 + 香菜（除非订单说"不要"）
- **香肠**：必放，不能"不要香肠"
- **烤冷面酱**：所有订单都必须有

#### 口味系统 (`FlavorType`)
- **甜辣** (`SWEET_SPICY`)：辣椒 + 白糖 + 烤冷面酱
- **酸甜辣** (`SOUR_SWEET_SPICY`)：醋 + 白糖 + 辣椒 + 烤冷面酱
- **酸甜** (`SOUR_SWEET`)：醋 + 白糖 + 烤冷面酱

#### 订单生成逻辑
- 根据 `timeProgress`（游戏时间进度）和 `customerIndex`（顾客位置）计算难度
- 难度越高，特殊要求越多（多放食材、多加调料、排除食材）
- 顾客数量动态调整（1→2→3，根据时间和差评率）

### 4. 打包和交付系统

#### 打包盒系统
- **3个独立打包盒**：`packingBox1`, `packingBox2`, `packingBox3`
- 每个打包盒对应一个顾客（从左到右：顾客1、2、3）
- 打包盒位置可在编辑器中自由调整

#### 打包流程
1. 制作好的食物可以**拖动**到打包盒
2. 检测逻辑：使用 `UITransform.hitTest()` 精确检测
3. 打包前检查：
   - 打包盒对应的顾客必须存在（`customer.node.active === true`）
   - 顾客必须有订单（`customer.order !== null`）
4. 食物放入打包盒后，**自动交付给对应顾客**
5. 打包盒使用后消失，可通过"摆放打包盒"按钮补充

#### 拖动检测
- 使用 `UITransform.hitTest(screenPos)` 进行精确检测
- 备用检测：矩形检测（0.5x节点大小）
- 支持精确拖动（鼠标保持在食物中心）

### 5. 评价和金币系统

#### 评价类型
- **超级好评** (`super`)：所有食材正确 + 流程正确 + 没烤焦
- **好评** (`good`)：所有食材正确 + 流程正确 + 烤焦了
- **差评** (`bad`)：食材错误或流程错误

#### 金币计算
- **基础价格**：6金币（无论评价如何）
- **额外鸡蛋**：每多1个蛋 +1金币
- **额外香肠**：每多1根香肠 +1金币
- **注意**：差评也会给金币（和好评一样多）

#### 评价影响
- **差评率**影响顾客数量（差评多 = 顾客少）
- **好评**增加顾客数量
- 评价历史存储在 `reviewHistory` 数组中（最多50条）

---

### 6. 时间系统

#### 游戏时间
- **游戏日长度**：8小时（12:00 PM - 8:00 PM）
- **时间转换**：2秒现实时间 = 1分钟游戏时间
- **时间管理器**：`TimeManager` 单例

#### 营业状态
- **开始营业前**：所有操作锁定（不能制作、打包、摆放打包盒）
- **开始营业**：按钮文字变为"结束营业"，可提前结束
- **营业中**：所有操作解锁
- **结束营业**：结算界面，显示总金币和评价统计

#### 时间进度 (`timeProgress`)
- 范围：0.0 - 1.0（0 = 12:00 PM，1.0 = 8:00 PM）
- 用于动态调整订单难度和顾客数量

---

### 7. 顾客系统

#### 顾客节点
- **3个顾客位置**：`Customer1`, `Customer2`, `Customer3`
- 每个顾客有：
  - `CustomerIcon` 子节点（临时图片，来的时候显示）
  - `orderLabel` 节点（订单文本，显示在头顶）

#### 顾客生成
- 根据 `timeProgress` 和 `badReviewRatio` 动态调整数量
- 初始：1个顾客（简单订单）
- 逐渐增加：2个顾客（1简单+1复杂）→ 3个顾客（都复杂）
- 差评率高时，顾客数量减少

#### 订单UI
- 订单文本显示在顾客头顶
- 格式：`📋 订单：\n加2个🥚\n2根🌭\n多放🧅洋葱\n不要🌿香菜\n甜辣`
- 订单完成后，UI自动清除

## 🐛 已知问题和解决方案

### 1. 坐标转换问题
- **问题**：拖动检测不准确
- **解决**：使用 `UITransform.hitTest()` 精确检测
- **位置**：`onFoodDragEnd()` 方法

### 2. 面饼位置问题
- **问题**：新面饼总是添加到第三个位置
- **解决**：使用 `Set` 跟踪已占用位置，循环查找空位
- **位置**：`placeDoughOnGrill()` 方法

### 3. 订单不更新问题
- **问题**：食物交付后订单不更新
- **解决**：在 `resetDailyState()` 中清除所有订单UI
- **位置**：`resetDailyState()` 方法

### 4. 游戏状态锁定问题
- **问题**：点击"开始营业"后所有操作仍锁定
- **解决**：实现延迟重试机制确保 `TimeManager` 回调注册成功
- **位置**：`ensureTimeManagerCallbacks()` 方法

## 🔄 游戏流程

### 完整流程
1. **游戏启动** → 显示"开始营业"按钮
2. **点击开始营业** → 时间开始流动，顾客逐渐出现
3. **制作食物**：
   - 点击面饼按钮 → 在烤架上生成面饼
   - 点击鸡蛋按钮 → 拿起鸡蛋
   - 点击面饼 → 添加鸡蛋
   - 再次点击面饼 → 翻面
   - 添加其他食材（洋葱、香菜、香肠、调料）
4. **打包交付**：
   - 拖动制作好的食物到打包盒
   - 自动交付给对应顾客
   - 获得金币和评价
5. **查看评价**：
   - 点击手机按钮
   - 查看评价历史
6. **结束营业**：
   - 点击"结束营业"按钮
   - 查看结算界面

## 💡 开发注意事项

### 1. 状态管理
- 面饼状态必须按顺序：RAW → EGG_ADDED → FLIPPED
- 不能跳过步骤（必须先加鸡蛋，再翻面，再加其他食材）

### 2. 订单验证
- 必须检查所有条件：鸡蛋数量、香肠数量、额外食材、调料、排除食材、口味
- 香肠是必放的，不能"不要香肠"

### 3. 时间系统
- `TimeManager` 是单例，需要确保初始化完成后再注册回调
- 使用 `ensureTimeManagerCallbacks()` 延迟重试机制

### 4. UI更新
- 订单UI是动态创建的，需要在订单完成或重置时清除
- 手机面板的评价列表也是动态创建的

### 5. 拖动检测
- 使用 `UITransform.hitTest()` 进行精确检测
- 需要将屏幕坐标转换为节点本地坐标
