## 问题分析

### 用户反馈
> "首先修复一下手机界面，目前的手机界面打开不太完善"

### 诊断结果

1. **响应式设计不足**
   - 无法自动适应不同屏幕尺寸
   - 手机和桌面显示效果相同
   - 字体大小固定

2. **布局问题**
   - 关卡按钮数量多时可能超出屏幕
   - 手机竖屏时无法完全显示
   - 没有为小屏幕优化

3. **节点位置问题**
   - Title位置可能偏低
   - LevelPanel位置需要调整
   - StartButton位置不是最优

## 修复方案

### 方案A：脚本层面优化 ✅

**文件**: `assets/Scripts/UI/MainMenuUI.ts`

#### 1. 添加屏幕尺寸检测

```typescript
private _screenWidth: number = 1280;
private _screenHeight: number = 720;

// 在onLoad中动态获取
const canvasTransform = canvas.parent.getComponent(UITransform);
this._screenWidth = canvasTransform.width;
this._screenHeight = canvasTransform.height;
```

**作用**：自动适应不同分辨率

#### 2. 实现响应式布局切换

```typescript
private optimizeLevelPanel() {
    const isMobile = this._screenWidth < 768;
    
    if (isMobile) {
        // 手机：竖直布局
        layout.type = Layout.Type.VERTICAL;
        layout.spacingY = 15;
    } else {
        // 桌面：水平布局
        layout.type = Layout.Type.HORIZONTAL;
        layout.spacingX = 20;
    }
}
```

**作用**：根据屏幕宽度自动切换布局模式

#### 3. 动态字体大小

```typescript
const fontSize = this._screenWidth < 768 ? 32 : 48;
this.titleLabel.fontSize = fontSize;
```

**作用**：手机显示32px，桌面显示48px

#### 4. 动态按钮宽度

```typescript
const btnWidth = isMobile ? this._screenWidth - 60 : 400;
transform.setContentSize(btnWidth, btnHeight);
```

**作用**：手机按钮适应屏幕宽度

### 方案B：场景调整 ✅

**文件**: `assets/Scenes/MainMenu.scene`

#### 节点位置优化

使用MCP工具调整了3个关键节点的Y坐标：

| 节点 | 修改前 | 修改后 | 原因 |
|------|-------|-------|------|
| Title | y=250 | y=300 | 显示在上方，距离更合理 |
| LevelPanel | y=0 | y=80 | 显示在中上方，避免与标题重叠 |
| StartButton | y=-200 | y=-280 | 显示在下方，留出足够空间 |

**效果**: 三个元素现在均匀分布在屏幕上
