## 🚀 快速开始指南

### 方法一：使用自动配置工具（推荐）⭐

1. **打开MainMenu场景**
   - 在Cocos Creator中打开 `assets/Scenes/MainMenu.scene`

2. **添加自动配置脚本**
   - 选中Canvas节点
   - 在属性检查器中点击"添加组件"
   - 搜索并添加 `MainMenuAutoSetup` 组件

3. **执行配置**
   - 在MainMenuAutoSetup组件中找到 `Execute` 属性
   - 点击勾选框，脚本会自动执行配置
   - 等待控制台输出 "🎉 主菜单配置完成！"

4. **绑定脚本属性**
   - 选中Canvas节点
   - 添加 `MainMenuUI` 组件
   - 按照下方属性列表绑定所有节点

5. **移除配置脚本**
   - 配置完成后，可以移除 `MainMenuAutoSetup` 组件

### 方法二：手动配置

请参考 `📘Cocos Creator配置指南.md` 文件，按步骤手动创建。

## 📝 MainMenuUI 属性绑定清单

配置完成后，需要在Canvas的MainMenuUI组件中绑定以下属性：

```
MainMenuUI 组件属性绑定：
├─ Title Label → Title节点的Label组件
├─ Start Btn → ButtonGroup/StartButton的Button组件
├─ Continue Btn → ButtonGroup/ContinueButton的Button组件
├─ Load Save Btn → ButtonGroup/LoadSaveButton的Button组件
├─ Settings Btn → ButtonGroup/SettingsButton的Button组件
├─ Quit Btn → ButtonGroup/QuitButton的Button组件
├─ Settings Panel → SettingsPanel节点
├─ Save List Panel → SaveListPanel节点
└─ Save List Content → SaveListPanel/Panel/ScrollView/View/Content节点
```

## ⚠️ 常见问题解决

### 问题1：按钮点击无响应
**原因**：MainMenuUI脚本的按钮属性未绑定
**解决**：
1. 选中Canvas节点
2. 检查MainMenuUI组件的所有Button属性是否已绑定
3. 确保绑定的是Button组件，不是节点

### 问题2：面板打开后看不见
**原因**：面板层级问题或默认为隐藏
**解决**：
1. 检查SettingsPanel和SaveListPanel的Active属性
2. 测试时可以先设为true查看是否存在
3. 检查面板的Position是否在屏幕范围内

### 问题3：存档列表为空但应该有存档
**原因**：SaveListUI的contentNode未正确绑定
**解决**：
1. 检查SaveListUI组件的Content Node属性
2. 应该绑定到 ScrollView/View/Content 节点
3. 确保Content节点有Layout组件

### 问题4：设置无法保存
**原因**：浏览器禁用LocalStorage或路径错误
**解决**：
1. 检查浏览器设置，允许LocalStorage
2. 查看控制台是否有LocalStorage相关错误
3. 使用浏览器开发者工具查看Application/Storage
