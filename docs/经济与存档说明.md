# 经济与存档说明

## 经济模型

当前规则：

- 每日结算只统计当天收益，不做额外扣费。
- 固定成本视为已在商店购买阶段结算（因此日结不再扣除）。
- 差评不产生罚金，仅用于统计与失败判定。

当前实现说明：

- 今日收益来自烹饪阶段的订单收入。
- 日结不弹出结算面板，直接进入下一天的商店阶段。
- 关卡通关条件仍使用原有逻辑（目标金额与差评限制）。

## 存档系统

位置：`assets/Scripts/Manager/SaveManager.ts`

### 存档内容

存档数据包含两部分：

- 基础信息（用于列表展示）：关卡、金币、统计信息
- `snapshot` 快照（用于恢复）：进度 + 钱包 + 关卡库存

快照结构（简化）：

```
snapshot:
  progress: GameProgress
  inventory:
    globalWallet
    unlockedLevel
    totalMoney
    dailyEarnings
    lastPlayDate
    currentLevel:
      levelId
      earnedMoney
      badReviewCount
      inventory[]
```

### 存档行为

- 自动存档：每天结算完成后自动保存。
- 自动存档触发：返回主菜单 / 退出游戏 / 应用切后台。
- 手动存档：烹饪阶段手机面板「存档」按钮。
- 读档入口：主菜单存档列表，支持旧存档兼容。

### 存档底层（文件 + 备份 + Steam Cloud）

位置：`assets/Scripts/Manager/SaveStorage.ts`

- 原生/桌面版使用文件存档，目录为 `getWritablePath()/saves/`。
- 每次覆盖写入前自动备份到 `getWritablePath()/saves/backups/`，默认保留最近3份。
- Steam Cloud 只需同步 `saves/` 目录即可实现跨设备存档。
- Web/编辑器预览自动回退到 `localStorage`，不影响编辑器测试。

### 调整建议

- 想要调整经济难度：直接修改 `EconomyConfig.ts` 的数值即可。
- 想要扩展存档内容：在 `snapshot` 中新增字段，并在 `InventoryManager` / `GameProgressManager` 的快照方法中同步。
