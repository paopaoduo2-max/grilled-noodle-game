# 特殊事件系统（模板库 1-3 关草案）

> 用途：在“实现事件系统”前先把模板库结构与内容定下来，方便你逐条调整。
>
> 共同规则：
> - 事件进度统一按“完成正常顾客订单并交付收款”计数（完成N单）。
> - 不允许出现“补货/购买食材”的事件内容。
> - 主奖励以金币/热度为主；主食材增减只做小幅波动（多数为失败惩罚）。
> - 券系统跨天保留，**3天内有效**（见各关“券事件”）。

## 时段建议（可在实现时配置）
- 早：开张后（偏轻量任务/宣传）
- 中：午高峰（偏冲刺/团单）
- 晚：晚高峰（偏更高难度冲刺）
- 夜：收尾（偏小任务/高风险高回报/券引导）
