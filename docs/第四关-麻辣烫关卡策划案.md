# 第四关：东北麻辣烫（自选称重 + 同锅并发）策划案 v1.0

> 目标：用于立刻进入实装的“可执行策划案”。本文把已讨论的设定完整收口，并补齐实现所需的关键规则/数值接口/交互细节。
>
> 关卡关键词：**自选称重、单次付款、同锅最多10份并发、多材质线程控品质、手动叫号交付、可雇佣服务员、街对面套圈小游戏（现金/食材礼包/终极大奖）**。

---

## 0. 设计前提（本关硬约束）

1. **购买阶段掠过**：不做“买菜/进货”玩法；进入关卡即开档营业。
2. **不做解冻/处理食材设定**：不引入解冻、切配等额外操作。
3. **顾客自选菜 → 称重计价 → 立即付款**：顾客付款后去等待区等待出餐。
4. **玩家负责烹饪**：一个大锅里**最多同时烹饪10份**（10个锅位/篮筐）。
5. **顾客系统重做但仅限本关**：不影响其他关卡的顾客/订单逻辑。
6. **顾客评价只影响热度（来客频率）**：本关**不做耐心值**、不做退款、无二次消费。
7. **小料台顾客自助**：玩家需要“维护/补充”小料台的可用性，但**小料不作为可掉落/可中奖的消耗品**；经济系统只围绕“食材”为消耗品展开。
8. **街对面小游戏是完整可玩内容**：不是给券；奖励只包含：**现金、食材礼包、终极大奖（大量现金 + 大量客流）**。

## 1. 关卡定位与玩家体验曲线

### 1.1 本关与前三关的继承点（参考第三关）
- **信息表达要清晰**：像第三关的“多个区块 + 进度条 + 状态提示”一样，本关要把“10份同锅并发”的复杂性做成**可读的优先级提示**，避免玩家觉得随机/看不懂。
- **质量影响节奏**：第三关用评价影响后续，本关保留闭环但简化：**评价只改变热度 → 只影响来客频率**。
- **刺激来自并发**：不靠“多步骤体力活”，靠“同时多份、多材质不同窗口”制造紧张。

### 1.2 核心情绪节奏（建议）
1) 开局稳定（30–45秒）：少量订单，建立“下锅→控品质→出锅→叫号”节奏  
2) 中期拉满（60–120秒）：锅位常驻 6–8 份，开始出现“多个线程同时报警”  
3) 后期疯狂（终局/事件）：锅位接近 10 份，交付栏堆积，套圈终极大奖可触发“客流暴涨”峰值

## 2. 总体流程（从顾客进店到离开）

### 2.1 顾客侧流程（无耐心、无二次消费）
1. 进入自选区（选菜）
2. 称重计价（按克/按份混合可选）
3. **立即付款**（金币入账）
4. 生成票号（`#001` 起）并进入等待区（只需要票号提示/头像，不需要耐心）
5. 玩家出餐后在“待交付栏”出现对应票号按钮
6. 玩家点击票号按钮“叫号交付”，顾客自动离场并生成评价（仅影响热度）

### 2.2 玩家侧核心循环（高并发）
自选区生成订单 → 订单入“待下锅队列” → 玩家把订单放入锅位（≤10） → 控火/搅拌/提篮等干预 → 出锅成品进入“待交付栏” → 手动点击票号交付

## 3. 订单与称重系统（顾客自选）

### 3.1 自选规则（推荐实现方式）
- 顾客从“菜品墙/菜盘池”中选择若干食材（可做权重偏好：叶菜/丸类/粉类/豆制品）。
- 每个食材有基础“重量区间”（例如：海带 40–80g，丸子 60–120g，粉 50–100g），顾客随机抽取重量。
- 订单结构建议拆成两层：
  - `OrderTicket`：票号、总重、总价、品质结果（出餐后才写入）
  - `OrderContent`：若干 `IngredientStack`（食材类型 + 重量 + 材质标签）

### 3.2 计价（按克为主）
- `总价 = Σ(食材单价(元/100g) * 重量(g) / 100) + 打包费(可选固定)`
- **付款时机**：称重后立即扣款/入账（本关无退款逻辑）。
- **为什么仍然刺激**：玩家会为了更高收益希望顾客选到“高价食材”，但高价食材往往更难控品质（窗口更苛刻/更易粘连/更易过火）。

### 3.3 食材库存与补货（本关主要支出来源）
> 本关“消耗品”只包含食材；小料台不走库存经济。

**库存扣减**
- 每份订单在“付款确认”时锁定所需食材重量；出锅/交付不再回滚（本关无退款）。
- 库存扣减建议按克：`stock[type] -= weight(g)`。

**自选池与缺货表现**
- 顾客自选仅从“库存足够的食材”里抽取；库存不足的食材在菜品墙上灰显/隐藏，避免生成不可完成订单。
- 缺货不是失败，只是降低客单价与组合复杂度（间接影响赚钱与热度上升速度）。

**补货（不做购买阶段，但要有花钱点）**
- 推荐做成“自动补货 + 可选急单补货”，不打断烹饪主循环：
  - 自动补货：当某食材低于阈值时自动补一箱并扣钱（提示“进货 -$X”）。
  - 急单补货（按钮）：立刻到货但更贵；用于高热度爆单期。
- 套圈摊的“食材礼包”奖励 = 直接增加库存，等价于“免费补货”。

## 4. 大锅并发烹饪系统（最多10份）

### 4.1 锅位与订单绑定
- 大锅内有 10 个“篮筐/锅位”，每个锅位只绑定 1 个订单票号。
- 锅位 UI 必须清晰显示：
  - 票号（大字）
  - 当前总体熟度条
  - 当前风险标签（欠熟/过火/粘连/散烂等）
  - “可出锅”提示（最佳窗口）

### 4.2 多材质“线程”设计（本关特效与玩法核心）
每份订单包含多个食材，不同食材归入不同材质线程，线程各自有窗口与风险。

**建议线程（可扩展）**
- `Leaf` 叶菜：窗口短，过火快（易“黄/蔫”）
- `Noodle` 粉/面：易粘连，需要周期性搅拌
- `Ball` 丸/肉：需要达到最低熟度（欠熟惩罚明显）
- `Root` 根茎：时间长但稳定（主要风险是过火变“面”）
- `Tofu` 豆制品：吸汤口感，过火会散

**实现简化建议（避免过度复杂）**
- 每份订单维护一个“主进度条”（整体熟度），同时维护 1–2 个“风险计时器”：
  - `粘连计时器`：粉面存在时启用，随时间上涨；搅拌可清零/衰减
  - `过火计时器`：整体熟度超过窗口后启用；火力越大增长越快

### 4.3 玩家可用的干预操作（简单但有深度）
1. **火力三档**（小/中/大）：影响整体加热速度与过火风险
2. **搅拌**（全锅动作）：降低/重置“粘连风险”，并对过火速度有轻微缓冲
3. **提篮离锅（挂边）**（单锅位）：把某一份暂时“离火”，用于错峰出锅/抢救过火

> 目标：把“多线程压力”集中在决策顺序与注意力分配，而不是复杂手势。

### 4.4 出锅与成品生成
- 当订单达到“可出锅窗口”时，锅位高亮，点击“出锅”生成 `DishResult`：
  - `票号`
  - `品质评级`（优/良/差）
  - `错误标签`（欠熟/过火/粘连/散烂）
  - `影响热度的分值`（见 6 章）
- 出锅后该锅位清空可再次放入新订单。

## 5. 交付系统（手动叫号 + 可雇佣服务员）

### 5.1 手动叫号（你已确认采用）
- 出锅后，成品进入“待交付栏”（显示**可点击票号按钮**）。
- 每个按钮必须由“真实已完成的成品”生成，避免空按钮：
  - 展示：`#018` + 品质小图标（优/良/差）
  - 点击：立即交付给对应票号顾客（顾客离场）
- 交付正确性：由于“按钮与成品一一对应”，**不存在交错交付**；刺激来自并发与交付栏堆积，而非找人/拖拽。

### 5.2 交付栏容量与堆积压力（建议）
- 交付栏可见按钮上限：`6`（可调）
- 超出进入隐藏队列（只显示“+N”提示），鼓励玩家及时交付
- 可选“保温衰减”机制（不涉及耐心）：
  - 成品在交付栏停留过久会**品质轻微下降**（只影响热度），逼迫玩家“做完就叫号”

### 5.3 服务员（可雇佣）

**设计目的**
- 本关的“忙”主要来自：同锅并发 + 材质线程报警 + 交付栏堆积。
- 服务员不是为了解决“交付正确性”，而是让玩家用金钱购买“操作带宽”，符合“越赚越多也花得越多、越刺激”的节奏。

**服务员职责边界（强约束）**
- ✅ 只处理：`待交付栏` → 自动点击叫号交付
- ❌ 不处理：下锅、控火、搅拌、提篮、出锅（避免变成“挂机”）

**雇佣与费用（建议）**
- 日薪：固定成本（随难度提升），开局选择是否雇佣；可中途加价临时雇佣（更贵）。
- 可选绩效：按交付量少量扣/加（让玩家在爆发期觉得“划算”）。

**能力等级（建议）**
- `Lv0` 无服务员：玩家手动点所有票号按钮
- `Lv1` 兼职：每 `1.5s` 自动交付 1 份（优先：保温衰减最快/最早完成的成品）；玩家可手动插队点击
- `Lv2` 全职：每 `0.9s` 自动交付 1 份；当交付栏≥4 时触发“连叫”，连续交付 2 份（冷却 6s）
- `Lv3` 金牌（后期可开）：每 `0.6s` 自动交付 1 份；连叫冷却 4s（费用显著提升）

**UI 表现**
- 服务员状态条：`忙碌/空闲`、下一次自动交付倒计时
- 升级按钮：只在非暂停状态可点；升级弹窗明确“日薪上涨 + 交付速度提升”

## 6. 评价 → 热度 → 来客频率（本关唯一“顾客系统反馈”）

### 6.1 评价来源（不含耐心、不影响钱）
评价在“交付瞬间”生成，仅影响热度。

**评价输入项（建议）**
- `品质评级`：优/良/差（来自熟度、粘连、过火等）
- `小料台可用性`：可用/缺料/脏乱（见 7 章）

> 不做：等太久、排队烦躁、退款、小费。

### 6.2 热度变化（建议规则）
- 交付 1 份后热度变化：
  - 优：`+2`
  - 良：`+1`
  - 差：`-2`
- 若交付时小料台处于“缺料/脏乱”状态：在上述基础上额外 `-1`
- 热度范围：`0 ~ 100`（沿用全局习惯）

### 6.3 来客频率（建议公式）
本关来客量要明显高于前几关，可用“热度→间隔”线性映射：
- `最慢间隔 maxInterval = 4.0s`（热度低也不断客）
- `最快间隔 minInterval = 0.9s`（热度高进入爆发）
- `spawnInterval = maxInterval - (heat / 100) * (maxInterval - minInterval)`

### 6.4 容量上限（避免无限堆积）
本关无耐心，因此必须有“硬上限”阻止无限生成。

- 等待区容量：`WAIT_CAP = 30`（建议默认，可配置）
- 待下锅队列上限：`QUEUE_CAP = 20`（避免 UI 爆炸）
- 规则：
  - 等待区满：停止生成新顾客（不扣热度）
  - 待下锅队列满：自选区暂时停流/排队提示（不扣热度）

## 7. 小料台（顾客自助 + 玩家维护，不作为经济消耗品）

### 7.1 设计边界（重要）
- 小料台的“芝麻酱/糖/葱花/香菜”不进入库存经济体系，不作为套圈奖品，不做购买消耗。
- 本关只把它当作：**额外分心点 + 影响热度的环境因素**。

### 7.2 可玩化方式（推荐：可用性条，不走库存）
每种小料有一个“可用性条”（0–100），顾客自助会降低；玩家可用短操作补满。

- 顾客取用：每位顾客随机消耗 1–2 种小料，每次 `-10~-25`
- 玩家补充：点击对应容器 → 播放 1.2s 补充动画 → 该小料回到 100
- 若任意一种小料 < 15：小料台进入“缺料”状态（交付时额外 -热度）

### 7.3 清洁/整齐（可选增强）
- 小料台累计使用次数会累积“脏乱值”，超过阈值进入“脏乱”状态（交付时额外 -热度）。
- 玩家可执行一次“擦拭”动作（1.5s）清零脏乱值。

## 8. 街对面小游戏：套圈摊（玩法与奖品分布）

> 目标：给玩家一个“用钱换高回报但高风险”的短局小游戏；其风险来自：小游戏期间锅不暂停，可能回到摊位就爆线程。

### 8.1 基础规则
- 单局时长：10–15 秒
- 默认圈数：3 个（可加购 5 个，价格更高）
- 小游戏进行中：大锅计时照常（不暂停）；可随时“提前返回”但本局作废/保留已命中奖励（实现二选一，推荐保留已命中）。

### 8.2 投掷操作（细化）
**一次投掷 = 按住 → 拖拽 → 松手**
- 按住圈进入瞄准态：
  - 显示“落点辅助圈”（只给大致，不给完美轨迹）
  - 显示“风/晃台提示”（见下）
- 拖拽决定：
  - 方向：投掷角度
  - 距离：投掷力度（力度条 0–100）
- 松手投掷：
  - 圈按抛物线飞出，落地后会滑动/弹跳（轻微物理）

**扰动机制（让它像地摊，不是纯瞄准）**
- `晃台`：摊主每 2–3 秒轻推台面一次，目标整体横向偏移少许（0.2–0.4 秒内完成）
- `微风`（可选）：屏幕顶部出现风向箭头，投掷中对落点产生小偏移（难度上限用）

### 8.3 命中判定（避免“擦边算不算”的争议）
- 圈的中心落在目标桩的判定圆内，并且速度 < 阈值，停留 ≥ 0.3s 视为命中
- 擦边弹出不算（给“差一点”的音效/特效即可）

### 8.4 奖品类型（仅现金/食材礼包/终极大奖）
**现金**
- 小现金：适合近排
- 中现金：适合中排
- 大现金：适合远排

**食材礼包（直接增加本关可用库存/上架量）**
- `素菜箱`：叶菜/菌菇/海带等（偏快熟线程）
- `主食箱`：粉/面/豆制品类（偏粘连线程）
- `荤菜箱`：丸子/肉片类（偏熟度线程）
- `综合箱`：三类混合（实用）
- `高级食材箱（稀有）`：高单价食材（提高称重客单价，但通常更难控品质/窗口更苛刻）

**终极大奖（你已确认）**
- `大量现金 + 大量客流`
  - 大量现金：一次性高额
  - 大量客流：立即把等待区填充到 `WAIT_CAP * 90%`（不使用耐心系统），并把热度瞬间拉到高位（例如 ≥80）

### 8.5 奖品分布（三层难度：近/中/远）
用“距离 + 目标大小 + 是否移动”作为隐性概率，不在 UI 上直接写概率（保持赌感）。

**推荐摆放（示例，5 列）**
- 远排（难，小桩/移动）：`[终极] [大现金] [高级食材箱] [大现金] [终极]`
- 中排（中，普通桩）：`[中现金] [综合箱] [荤菜箱] [综合箱] [中现金]`
- 近排（易，大桩）：`[小现金] [素菜箱] [主食箱] [素菜箱] [小现金]`

**目标大小建议**
- 近排：半径 48
- 中排：半径 36
- 远排：半径 26（终极桩可更小或移动更快）

### 8.6 消费与回报（建议）
- 3圈价格：低门槛（让玩家敢尝试）
- 5圈加购：高溢价（符合“越赚越花”）
- 期望回报：长期期望值略低于投入（保持赌感），但“终极大奖”让玩家愿意冒险

## 9. 数值与配置清单（便于程序落地）

> 建议全部做成 `Level4Config` 可调参数，避免写死。

### 9.1 容量与节奏
- `POT_SLOT_CAP = 10`
- `WAIT_CAP = 30`
- `QUEUE_CAP = 20`
- `DELIVERY_VISIBLE_CAP = 6`
- `spawnInterval(min,max) = (0.9s, 4.0s)`

### 9.2 热度与评价
- 热度范围：`0..100`
- 优/良/差对热度：`+2 / +1 / -2`
- 小料台缺料/脏乱额外：`-1`
- 终极大奖客流：等待区填充至 `WAIT_CAP*0.9`，热度设为 `>=80`

### 9.3 服务员
- `Lv1` 交付间隔 `1.5s`
