## 特殊事件文案覆写（可手写）

目标：把【特殊事件】的剧情描述/发送者/选项文案放到一个可编辑文件里，你后期随时修改，游戏弹窗会按覆写内容显示（未覆写的字段仍走代码默认模板/兜底）。

### 入口文件

- 覆写表：`assets/Resources/SpecialEvents/special_event_overrides.json`

> 这是 Resources 资源，运行时通过 `resources.load('SpecialEvents/special_event_overrides')` 读取。

### 文件结构（最小示例）

```json
{
  "version": 1,
  "events": {
    "L1_M01": {
      "description": "你手写的剧情……",
      "sender": { "name": "隔壁摊小哥", "icon": "📢", "role": "消息" },
      "optionA": { "text": "选项A文案" },
      "optionB": { "text": "选项B文案" }
    }
  }
}
```

### 支持覆写的字段

- `description`：事件正文（支持换行，用 `\\n`）
- `sender`：发送者信息
  - `name`：名称
  - `icon`：图标/emoji
  - `role`：角色/身份
- `optionA` / `optionB`：
  - `text`：选项文案
  - `emoji`：可选（如果 UI 有单独显示 emoji 的逻辑会用到；没有则不影响）

没写的字段会保留模板原值。

### 如何找到 eventId（例如 L1_M01）

事件 id 来自 `assets/Scripts/Game/SpecialEvents/SpecialEventTemplates.ts` 中的模板定义（如 `makeEvent('L1_M01', ...)`）。

### 修改后如何生效

- 事件弹窗出现时会尝试在编辑器/预览环境下节流重载覆写表；如果你发现修改后没立刻更新，通常是 Creator 的资源缓存导致：
  - 先关掉预览再重新 Preview 一次（最稳）
  - 或者等下一次事件弹窗触发再观察

### 代码接入位置（方便你后续查）

- 读取/合并逻辑：`assets/Scripts/Game/SpecialEvents/SpecialEventTextOverrides.ts`
- 使用位置：`assets/Scripts/Game/SpecialEvents/SpecialEventSystem.ts`（弹窗展示前 apply）
