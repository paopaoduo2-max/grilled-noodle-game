## 🎯 关键配置解释

### 为什么Canvas的cameraComponent要留空？

**正确做法：**
```yaml
Canvas:
  cameraComponent: null  # 留空
```

**原因：**
- Cocos Creator 会自动查找场景中的Camera
- 手动设置容易出错
- null值让系统自动处理

**错误做法：**
```yaml
Canvas:
  cameraComponent: <某个错误的引用>  # ❌
```

**结果：**
```
this._cameraComponent._createCamera is not a function
```

### 为什么Canvas Layer必须是UI_2D？

**Layer和Visibility关系：**

```
Camera Visibility = UI_2D (33554432) | DEFAULT (1073741824)
                  = 1107296256

Canvas Layer = UI_2D (33554432)

渲染条件：
(Canvas Layer & Camera Visibility) != 0
(33554432 & 1107296256) = 33554432 ≠ 0  ✅ 可见
```

**如果Canvas Layer = DEFAULT：**
```
(1073741824 & 1107296256) = 1073741824 ≠ 0  ✅ 也可见
```

**但标准做法：**
- Canvas和UI元素应该在 **UI_2D 层**
- 这是Cocos Creator的惯例和最佳实践

## 🔧 修复MainMenuUI脚本

我也修复了脚本中的事件监听问题：

**修复前（可能出错）：**
```typescript
private setupButtons() {
    if (this.startBtn) {
        this.startBtn.node.on(Button.EventType.CLICK, this.onStartGame, this);
    }
}
```

**修复后（更安全）：**
```typescript
private setupButtons() {
    console.log('[MainMenuUI] setupButtons开始');
    
    if (this.startBtn && this.startBtn.node) {
        console.log('[MainMenuUI] 绑定startBtn事件');
        this.startBtn.node.on(Button.EventType.CLICK, this.onStartGame, this);
    } else {
        console.warn('[MainMenuUI] startBtn或startBtn.node不存在');
    }
    
    console.log('[MainMenuUI] setupButtons完成');
}
```

- ✅ 检查 `startBtn.node` 是否存在
