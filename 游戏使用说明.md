## 项目概述

**东北料理王** 是一款类似"沙威玛传奇"的2D模拟经营游戏。玩家经营一家东北料理摊位，通过准备食材、制作美食、服务客户来赚取金币并通过关卡。

### 游戏特点
- **两阶段玩法**：准备阶段购买食材 + 制作阶段服务客户
- **关卡系统**：7大关卡，逐步解锁
- **第一关**：制作烤冷面
- **多平台支持**：主要面向Steam，未来支持移动端

## 快速开始

### 1. 打开项目
1. 启动 Cocos Creator 3.x
2. 打开项目目录：`D:\xm\NewProject_1`
3. 等待项目加载完成

### 2. 编译脚本
Cocos Creator 应该会自动编译 TypeScript 脚本。如果没有，请：
- 点击菜单：`开发者 -> 编译脚本`
- 检查控制台是否有错误

### 3. 设置启动场景
1. 打开 `MainMenu.scene`
2. 点击菜单：`项目 -> 项目设置 -> 预览运行设置`
3. 设置"启动场景"为 `MainMenu`

### 4. 运行预览
- 点击编辑器顶部的"预览"按钮（▶️）
- 或按快捷键 `Ctrl+P` (Windows) / `Cmd+P` (Mac)

## 游戏流程

### 完整游戏流程

```
MainMenu (主菜单)
    ↓ 选择关卡 + 点击开始
PrepareScene (准备阶段)
    ↓ 购买食材 + 点击开始营业
CookingScene (制作阶段)
    ↓ 服务客户 + 时间结束/手动结束
ResultScene (结算界面)
    ↓ 查看结果 + 选择下一步
MainMenu / PrepareScene / 下一关
```

### 准备阶段（PrepareScene）

**目标：** 使用初始金币购买足够的食材

1. **初始状态**
   - 显示关卡名称
   - 显示当前金币（例如：50金币）
   - 显示准备时间倒计时（例如：60秒）

2. **玩家操作**
   - 查看食材价格和库存
   - 点击"购买"按钮购买食材
   - 规划食材数量（够用即可，省钱）

3. **食材列表（第一关 - 烤冷面）**
   - 面饼：2金币/个
   - 鸡蛋：1金币/个
   - 香肠：3金币/个
   - 香菜：1金币/个
   - 酱料：2金币/个

4. **制作配方**
   ```
   烤冷面 = 面饼x1 + 鸡蛋x1 + 香肠x1 + 香菜x1 + 酱料x1
   售价：15金币
   制作时间：5秒
   ```

5. **进入下一阶段**
   - 点击"开始营业"按钮
   - 或等待准备时间结束（自动进入）

### 制作阶段（CookingScene）

**目标：** 
- 赚取目标金额（第一关：200金币）
- 完成目标客户数（第一关：15位客户）

1. **客户系统**
   - 客户会自动出现（每8秒一位）
   - 最多同时存在5个订单
   - 每个客户都有耐心值（时间条）

2. **客户类型**
   - **普通客户** (蓝色)
     - 耐心：30秒
     - 小费倍率：1.0x
   - **VIP客户** (金色)
     - 耐心：45秒
     - 小费倍率：2.0x
   - **着急的客户** (红色)
     - 耐心：15秒
     - 小费倍率：1.5x

3. **制作流程**
   - 点击客户的"开始制作"按钮
   - 系统自动消耗食材
   - 等待烹饪完成（5秒）
   - 自动交付给客户，获得金币

4. **评分系统**
   - 基础评分：100分/份
   - 速度加成：剩余耐心越多，评分越高
   - 超时惩罚：客户离开，不获得金币

5. **同时烹饪**
   - 最多同时制作3份
   - 合理安排制作顺序
   - 优先服务着急的客户

6. **结束营业**
   - 营业时间结束（180秒）
   - 或点击"结束营业"按钮

### 结算阶段（ResultScene）

**显示内容：**
- ✅/❌ 通过/失败标题
- 赚取金币 vs 目标金币
- 完成客户数 vs 目标客户数
- 总评分
- 详细统计：
  - 完成订单数
  - 超时订单数
  - 总收入
  - 总支出
  - 净利润

**按钮选项：**
- **重试**：重新挑战当前关卡
- **下一关**：进入下一关（仅通过时显示）
- **返回菜单**：回到主菜单

## 核心系统说明

### 1. GameManager（游戏管理器）

**职责：** 全局游戏状态管理、场景切换、数据持久化

**单例访问：**
```typescript
const gameManager = GameManager.Instance;
```

**主要方法：**
```typescript
// 开始关卡
gameManager.startLevel(levelId);

// 购买食材
gameManager.buyIngredient(IngredientType.NOODLE, 5);

// 使用食材
gameManager.useIngredient(IngredientType.EGG, 1);

// 完成订单
gameManager.completeOrder(earnMoney, score);

// 进入下一阶段
gameManager.enterPreparePhase();
gameManager.enterCookingPhase();
gameManager.enterResultPhase();
```

**数据访问：**
```typescript
gameManager.playerMoney          // 当前金币
gameManager.currentLevel         // 当前关卡数据
gameManager.completedCustomers   // 完成客户数
gameManager.totalScore           // 总评分
```

### 2. CustomerManager（客户管理器）

**职责：** 生成客户、管理订单队列、处理订单超时

**初始化：**
```typescript
customerManager.init(recipes, spawnInterval, maxOrders);
// recipes: 可制作的菜品
// spawnInterval: 客户生成间隔（秒）
// maxOrders: 最大同时订单数
```

**事件监听：**
```typescript
node.on('customer-arrived', (order) => {
    // 新客户到达
});

node.on('order-completed', (data) => {
    // 订单完成：{ order, money, score }
});

node.on('order-timeout', (order) => {
    // 订单超时
});
```

### 3. CookingSystem（烹饪系统）

**职责：** 管理烹饪任务、食材消耗、烹饪进度

**初始化：**
```typescript
cookingSystem.init(maxSlots);
// maxSlots: 最大同时烹饪位数
```

**主要方法：**
```typescript
// 开始烹饪
const result = cookingSystem.startCooking(orderId, recipe);
if (result.success) {
    console.log('烹饪已开始', result.taskId);
}

// 完成任务
cookingSystem.finishTask(taskId);

// 获取活跃任务
const tasks = cookingSystem.getActiveTasks();
```

**事件监听：**
```typescript
node.on('cooking-started', (task) => {
    // 烹饪开始
});

node.on('cooking-done', (task) => {
    // 烹饪完成
});
```

### 4. GameConfig（游戏配置）

**职责：** 存储所有游戏静态数据

**关卡配置：**
```typescript
GameConfig.LEVELS[0]  // 第一关数据
// 包含：关卡名、描述、目标、时间、初始金币等
```

**食材配置：**
```typescript
GameConfig.INGREDIENTS_CONFIG[IngredientType.NOODLE]
// 包含：名称、价格、颜色等
```

**菜谱配置：**
```typescript
GameConfig.RECIPE_GRILLED_COLD_NOODLE
// 包含：所需食材、制作时间、售价等
```

## UI完善指南

由于当前没有美术资源，所有UI元素都使用纯色Sprite和文字。以下是推荐的颜色方案：

```typescript

new Color(220, 200, 180, 255)

// 准备阶段背景：浅蓝色
new Color(180, 220, 240, 255)

// 制作阶段背景：橙黄色（热闹的厨房）
new Color(255, 220, 150, 255)

// 结算背景：深灰色
new Color(100, 100, 120, 255)
```

- 鸡蛋：#FFD700（金黄色）
- 香肠：#D2691E（褐色）
- 香菜：#90EE90（浅绿色）
- 洋葱：#DDA0DD（紫色）
- 酱料：#8B4513（深褐色）
- 醋：#CD853F（棕色）
- 辣椒：#FF4500（红色）

#### 按钮颜色
- 普通按钮：(100, 200, 255) 蓝色
- 确认按钮：(100, 200, 100) 绿色
- 取消按钮：(200, 100, 100) 红色

```typescript
        }
```

```typescript
    sys.localStorage.setItem('game_data', JSON.stringify(data));
}
```

## 常见问题

### Q1: 场景跳转失败？
**A:** 检查以下几点：
- 确保场景名称正确（区分大小写）
- 在"项目设置 -> 构建发布"中将所有场景添加到"参与构建场景"列表
- 检查控制台是否有错误信息

### Q2: 脚本组件添加失败？
**A:** 
- 确保脚本没有语法错误
- 重新编译脚本：菜单 -> 开发者 -> 编译脚本
- 刷新资源：右键 Assets 文件夹 -> 刷新

### Q3: UI节点引用为空？
**A:**
- 确保在属性检查器中正确绑定了引用
- 使用 `console.log()` 调试检查节点是否存在
- 确认节点路径和名称正确

### Q4: 客户不出现？
**A:**
- 检查CustomerManager是否正确初始化
- 确认recipes数组不为空
- 查看控制台日志，确认客户生成逻辑运行

### Q5: 食材购买后没有扣钱？
**A:**
- 检查GameManager是否正确初始化
- 确认startLevel()方法被调用
- 使用调试器查看_playerMoney变量

### Q6: 如何调试游戏？
**A:**
1. 使用 `console.log()` 输出信息
2. Chrome DevTools：预览时按 F12
3. 在关键位置添加断点
4. 检查Cocos Creator控制台

### Q7: 如何修改关卡难度？
**A:** 在 `GameConfig.ts` 中修改关卡数据：
```typescript
targetMoney: 200,        // 增加目标金额
targetCustomers: 15,     // 增加目标客户
prepareTime: 60,         // 减少准备时间
cookingTime: 180,        // 减少营业时间
initialMoney: 50         // 减少初始金币
```

### Q8: 如何添加更多食材购买数量选项？
**A:** 修改 `PreparePhaseUI.ts` 中的创建方法，添加多个按钮：
```typescript
// 购买 x1, x5, x10
const buyAmounts = [1, 5, 10];
buyAmounts.forEach(amount => {
    // 创建按钮...
});
```

### Q9: 游戏太难/太简单？
**A:** 调整以下参数：
- 客户生成间隔（`_spawnInterval`）
- 客户耐心值（`GameConfig.CUSTOMER_TYPES`）
- 食材价格（`GameConfig.INGREDIENTS_CONFIG`）
- 菜品售价（`RecipeData.price`）
- 制作时间（`RecipeData.cookTime`）

### Q10: 如何打包发布？
**A:**
1. 点击菜单：项目 -> 构建发布
2. 选择平台（Web Mobile、Windows等）
3. 设置发布路径
4. 点击"构建"
5. 构建完成后点击"发布"
