## ⚠️ 可能遇到的小问题

### 问题1：UI显示不完整
**原因：** 某些节点名称不匹配

**解决：**
```
打开浏览器控制台（F12），查看错误信息
脚本会尝试自动查找节点，找不到的会显示 null
```

### 问题2：食材不显示
**这是正常的！**

食材是在PreparePhaseUI的`start()`方法中**动态生成**的，不是预先创建的。

当你进入准备阶段后，脚本会：
1. 查看当前关卡需要哪些食材
2. 为每种食材创建UI元素（纯色方块+文字）
3. 自动添加到IngredientPanel中

### 问题3：客户不出现
**请等待8秒！**

客户管理器会每隔8秒生成一个新客户，最多同时5个订单。

你可以修改生成间隔：
```typescript
// 在 CookingPhaseUI.ts 中
customerManagerComp.init(
    recipes,
    5,  // 改小这个数字，比如改成3，就是3秒一个客户
    5
);
```

## ✨ 新增功能

### 自动节点查找
所有UI脚本现在都会：
1. 首先检查属性是否已绑定
2. 如果没有绑定，自动通过路径查找节点
3. 找到后自动获取对应组件
4. 完全无需手动绑定！

### 查找路径示例
```typescript
// MainMenuUI
'Title' → 获取Title节点的Label组件
'LevelPanel' → 获取LevelPanel节点
'StartButton' → 获取StartButton节点的Button组件

// PreparePhaseUI
'IngredientPanel' → 食材面板
'TopPanel/LevelNameLabel' → 关卡名称Label
'TopPanel/MoneyLabel' → 金币Label
'TopPanel/TimerLabel' → 计时器Label
'StartCookingBtn' → 开始营业按钮

// CookingPhaseUI
'OrderPanel' → 订单面板
'CookingPanel' → 烹饪面板
'TopInfoPanel/MoneyLabel' → 金币Label
'TopInfoPanel/TimerLabel' → 时间Label
'TopInfoPanel/ScoreLabel' → 评分Label
'TopInfoPanel/CustomerCountLabel' → 客户数Label
'EndDayBtn' → 结束营业按钮

// ResultUI
'ResultPanel/TitleLabel' → 标题Label
'ResultPanel/MoneyLabel' → 金币Label
'ResultPanel/TargetMoneyLabel' → 目标金币Label
... 等等
```
