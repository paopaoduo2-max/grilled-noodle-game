## 📊 技术说明

### 为什么需要Camera？

**Cocos Creator 3.x 的渲染管线：**
```
场景节点 → Camera → 渲染器 → 屏幕
```

没有Camera，渲染管线就断了，什么都看不到。

### Camera关键属性解释

| 属性 | 值 | 说明 |
|-----|---|------|
| **clearFlags** | 7 (SOLID_COLOR) | 每帧用纯色清除屏幕 |
| **projection** | 0 (ORTHO) | 正交投影，适合2D游戏 |
| **priority** | 1 | 渲染优先级 |
| **clearColor** | (51,51,51,255) | 深灰色背景 |
| **near** | 0 | 近裁剪面 |
| **far** | 1000 | 远裁剪面 |

### clearFlags 选项说明

- **0 (DONT_CLEAR)**: 不清除 - 会留下上一帧的内容
- **6 (DEPTH_ONLY)**: 只清除深度 - 2D游戏不适用
- **7 (SOLID_COLOR)**: ✅ 用纯色清除 - 2D UI最佳选择
- **14 (SKYBOX)**: 天空盒 - 3D游戏使用

### projection 选项说明

- **0 (ORTHO)**: ✅ 正交投影 - 2D游戏标准配置
- **1 (PERSPECTIVE)**: 透视投影 - 3D游戏使用

## 📚 相关概念

### Cocos Creator 3.x Camera系统

**Camera是渲染管线的核心组件：**

```typescript
// Camera的主要职责
1. 定义视口范围（Viewport）
2. 设置投影方式（正交/透视）
3. 控制渲染顺序（Priority）
4. 决定哪些层可见（Visibility）
5. 清除屏幕缓冲（Clear Flags）
```

**多Camera场景：**
```
Scene
  ├─ Background Camera (Priority: 0)
  │   └─ 渲染背景层
  ├─ Game Camera (Priority: 1)
  │   └─ 渲染游戏对象
  └─ UI Camera (Priority: 2)
      └─ 渲染UI层
```

**我们的简单2D游戏只需要一个Camera。**

### 2D vs 3D Camera配置对比

| 特性 | 2D游戏 (我们的设置) | 3D游戏 |
|------|-------------------|--------|
| Projection | ORTHO (正交) | PERSPECTIVE (透视) |
| Clear Flags | SOLID_COLOR | SKYBOX / SOLID_COLOR |
| Position Z | 任意（不影响视角） | 影响视角距离 |
| FOV | 无（正交无此属性） | 45° ~ 90° |
